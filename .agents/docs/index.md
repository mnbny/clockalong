## Documentation Style

Docs are for discovery and non-obvious project patterns. Do not document details that are self-explanatory in code,
easy to find with file search, or only describe the current implementation mechanically. Prefer high-level maps,
ownership boundaries, cross-file contracts, and conventions that help future agents know where to look and what
assumptions to preserve.
